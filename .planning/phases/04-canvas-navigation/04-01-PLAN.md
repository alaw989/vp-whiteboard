<task type="auto">
  <name>Create useViewport composable with zoom/pan state management</name>
  <files>composables/useViewport.ts</files>
  <action>
Create a new composable `composables/useViewport.ts` that:

1. Accepts ViewportOptions interface:
   - stageRef: Ref<any> - Reference to v-stage component
   - containerRef: Ref<HTMLDivElement | null> - Container for size calculations
   - minZoom?: number (default 0.1)
   - maxZoom?: number (default 5.0)
   - onViewportChange?: (viewport: ViewportState) => void - Optional callback for changes

2. Exports reactive state:
   - viewport: Ref<ViewportState> with x, y, zoom properties
   - stageConfig: ComputedRef returning Konva stage config (scaleX, scaleY, x, y, draggable)

3. Implements handleWheel function for pointer-relative zoom:
   - Gets pointer position from stage.getPointerPosition()
   - Calculates scale factor: 1.1 for in, 1/1.1 for out
   - Clamps new scale between minZoom and maxZoom
   - Adjusts position to zoom toward pointer: newPos = pointer - (pointer - oldPos) * (newScale / oldScale)
   - Updates viewport ref and calls onViewportChange callback

4. Implements pan control functions:
   - startPan(): Sets stage.draggable(true)
   - stopPan(): Sets stage.draggable(false) and captures final position

5. Implements programmatic zoom functions:
   - zoomIn(): Increments zoom by 1.1x toward center
   - zoomOut(): Decrements zoom by 1/1.1x toward center
   - resetZoom(): Resets to zoom: 1, x: 0, y: 0

Follow existing composable patterns from useSelection.ts and useDocumentLayer.ts:
- Use readonly() for exported refs to prevent external mutation
- Return functions and computed properties for reactive updates
- Clean up any side effects in onUnmounted if needed

DO NOT:
- Do not use CSS transforms - only Konva stage.scale/position
- Do not sync to Yjs directly in this composable (Plan 03 will add sync)
- Do not add touch gestures yet (Plan 03 may extend for mobile)
  </action>
