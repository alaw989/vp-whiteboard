<objective>
Synchronize viewport state (x, y, zoom) across all connected users using Yjs. When one user zooms or pans, all users see the same viewport. Implement conflict avoidance using "last updated by" tracking to prevent infinite loops.

Purpose: Enable collaborative navigation where engineers and clients can maintain a shared view of the drawing. Viewport sync uses the existing yMeta Map (separate from yElements) to avoid polluting undo history with navigation changes.

Output: Viewport state synchronized via Yjs yMeta Map with conflict-avoidance and debouncing.
</objective>

2. In useCollaborativeCanvas.ts, add viewport sync functions after importState (around line 209):

   getViewport():
   - Returns yMeta.get('viewport') as SharedViewportState | undefined
   - Returns { x: 0, y: 0, zoom: 1 } if not set

   syncViewport(viewport: ViewportState):
   - Accepts current user's viewport state
   - Wraps update in ydoc.transact() with userId origin
   - Sets yMeta.set('viewport', {
       ...viewport,
       lastUpdatedBy: userId,
       timestamp: Date.now(),
     })
   - Only calls when viewport actually changes (debounce)

   observeViewport(callback: (viewport: SharedViewportState) => void):
   - Sets up yMeta.observe() listener
   - Filters for 'viewport' key changes
   - Calls callback with remote viewport if lastUpdatedBy !== userId
   - Returns cleanup function to remove observer

DO NOT:

- Do not use separate WebSocket for viewport (Yjs handles sync)
- Do not auto-apply remote viewport in this function (let composable decide)
