Implement CRDT memory garbage collection to prevent unbounded memory growth during long sessions.

Purpose: Yjs accumulates tombstone records for deleted items, and the UndoManager retains full history. Over long sessions, this causes memory growth. Periodic garbage collection keeps memory usage bounded.

Output: Periodic document compaction with undo history clearing, configurable interval, and exposed GC control functions.
</objective>

       ```typescript
       function compactDocument() {
         // Compaction invalidates undo history - log warning
         console.log('[CRDT] Compacting document - undo history will be cleared')

         ydoc.transact(() => {
           // Strategy: Clear undo manager history
           // This removes references to deleted items
           if (undoManager.canUndo()) {
             undoManager.clear()
           }

           // Note: We don't compact yElements array itself because:
           // - It would break collaborative context for other users
           // - Active users may have references to old elements
           // - Memory impact is minimal compared to tombstones
         }, 'compaction')
       }
       ```

    2. Create startGarbageCollection function:
       ```typescript
       let gcInterval: ReturnType<typeof setInterval> | null = null

       function startGarbageCollection(intervalMs: number = 10 * 60 * 1000) {
         // Default: 10 minutes
         if (gcInterval) {
           clearInterval(gcInterval)
         }

         gcInterval = setInterval(() => {
           compactDocument()
         }, intervalMs)

         return () => {
           if (gcInterval) {
             clearInterval(gcInterval)
             gcInterval = null
           }
         }
       }
       ```
