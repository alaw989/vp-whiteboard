  <files>composables/useCollaborativeCanvas.ts</files>
  <action>
Modify /home/deck/Sites/vp-whiteboard/composables/useCollaborativeCanvas.ts:

1. After existing yElements declaration, add:
   const yActiveStrokes = ydoc.getMap<Record<string, [number, number, number][]>>('activeStrokes')
2. Create local reactive ref for observing remote active strokes:
   const activeStrokes = ref<Record<string, [number, number, number][]>>()
3. Implement startActiveStroke(strokeId: string) function:
   - Call ydoc.transact(() => { yActiveStrokes.set(strokeId, []) }, userId)
   - This creates empty array for this user's active stroke
4. Implement broadcastStrokePoint(strokeId: string, point: [number, number, number]) function:
   - Call ydoc.transact(() => {
       const existing = yActiveStrokes.get(strokeId) || []
       yActiveStrokes.set(strokeId, [...existing, point])
     }, userId)
   - Appends point to existing array
5. Implement endActiveStroke(strokeId: string, element: CanvasElement) function:
   - Call ydoc.transact(() => {
       yActiveStrokes.delete(strokeId)
       yElements.push([element])
     }, userId)
   - Removes from active strokes and adds to permanent elements
6. Observe yActiveStrokes for remote changes:
   - yActiveStrokes.observe((event) => {
       event.changes.keys.forEach((key) => {
         const strokeId = key as string
         const points = yActiveStrokes.get(strokeId)
         if (points && points.length > 0) {
           activeStrokes.value[strokeId] = points
         } else {
           delete activeStrokes.value[strokeId]
         }
       })
     })
7. Export new functions and ref along with existing exports:
   - startActiveStroke, broadcastStrokePoint, endActiveStroke, activeStrokes
   - Keep all existing exports (ydoc, yElements, etc.)

IMPORTANT: Filter out own strokes from activeStrokes observation - only render remote users' in-progress strokes.
</action>
